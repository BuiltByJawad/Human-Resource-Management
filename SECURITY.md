# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously and appreciate your efforts to responsibly disclose any vulnerabilities you find.

### How to Report a Security Issue

Please report security issues by emailing our security team at [security@hrm.com](mailto:security@hrm.com). We will respond within 48 hours to acknowledge your report.

**Please do not report security vulnerabilities through public GitHub issues or discussions.**

### What to Include in Your Report

- Description of the vulnerability
- Steps to reproduce the issue
- Impact of the vulnerability
- Any proof-of-concept code or exploit
- Your name and affiliation (if any) for credit

### Our Security Process

1. Your report will be acknowledged within 48 hours
2. We will investigate the issue and determine the impact
3. We will develop a fix and test it
4. We will release a security update
5. We will publicly acknowledge your contribution (unless you prefer to remain anonymous)

### Security Updates

Security updates will be released as patch versions (e.g., 1.0.1, 1.0.2) with details in the changelog.

## Security Best Practices

### For Users
- Keep your installation up to date
- Use strong, unique passwords
- Enable 2FA for all user accounts
- Regularly review user access and permissions
- Keep your server and dependencies updated

### For Developers
- Follow secure coding practices
- Never commit sensitive data to version control
- Use environment variables for configuration
- Keep dependencies updated
- Perform security audits regularly

## Responsible Disclosure Policy

We follow a 90-day disclosure timeline for security issues:
- Day 0-30: Investigation and patch development
- Day 30-60: Testing and verification
- Day 60-90: Coordinated disclosure with patch release

## Security Features

- Data encryption at rest and in transit
- Regular security audits
- Automated dependency scanning
- Rate limiting and request validation
- Security headers and CSP
- Regular security training for developers

For security-related inquiries, please contact [security@hrm.com](mailto:security@hrm.com)
